🗂️ 3. Database Design (tối ưu + index đầy đủ)
3.1 users
id (PK)
username (unique, index)
password_hash
role (index) -- admin / worker / user
created_at
3.2 customers (hộ dân)
id (PK)
name
address
user_id (FK -> users.id, index) -- liên kết khách
created_at

👉 1 user = 1 hộ (có thể mở rộng sau)

3.3 meter_readings (chỉ số nước)
id (PK)
customer_id (FK, index)
month (YYYY-MM, index)
reading
image_url
created_by (FK -> users.id)
created_at

UNIQUE(customer_id, month)

👉 Index quan trọng:

(customer_id, month)
created_at
3.4 payments (giao dịch thanh toán)
id (PK)
customer_id (FK, index)
amount
status (pending, success, failed, index)
method (napas, sepay)
transaction_ref (unique, index)
created_at
paid_at
3.5 payment_logs (audit cực quan trọng)
id (PK)
payment_id (FK, index)
raw_request (jsonb)
raw_response (jsonb)
status
created_at

👉 Dùng để:

Debug
Đối soát ngân hàng
Tránh dispute
3.6 system_logs
id (PK)
user_id
action
metadata (jsonb)
created_at (index)
⚡ Index chiến lược (rất quan trọng)
users.username (unique)
meter_readings(customer_id, month) (unique)
payments.transaction_ref (unique)
payments.status
logs.created_at
